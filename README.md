# نظام أتمتة قناة يوتيوب — MVP (مرحلة الشورتس فقط)

نظام إنتاج ونشر تلقائي بالكامل لشورتس معلوماتية موجّهة لجمهور أمريكي، مبني
على ترندات يوتيوب الفعلية، بدقة 1080x1920 مضمونة. **المسار النشط الوحيد
حالياً هو الشورت** (`shorts_pipeline.yml`) — كود الفيديو الطويل (5 دقائق)
موجود وجاهز بالكامل لكن معطّل بالجدولة عمداً حتى تثبت نتائج الشورتس لأسابيع.

## ما الجديد بهذا التحديث

- ✅ دقة الشورت مضمونة 1080x1920 (طبقتين: Remotion Composition + فحص ffprobe قبل الرفع)
- ✅ هوامش أمان حقيقية للنص (`SHORTS_SAFE_ZONE` بـ `config.py`): يمين 14%
  (أزرار الواجهة)، أسفل 26% (عنوان/اسم القناة)، يسار 4% (توازن بصري)
- ✅ حركة Ken Burns مخفّضة الشدة بالشورت حتى لا يُدفع الموضوع الرئيسي بالصورة
  تحت أزرار الواجهة
- ✅ `scripts/content_policy.py`: قائمة حظر شاملة (خمر، مخدرات، عري، قمار،
  محتوى جنسي، + حساسية سياسية/إخبارية) مطبّقة كطبقة **حتمية** (hard block)
  بـ 3 نقاط: اختيار الترند، تقييم Quality Gate، وبرومبت كتابة السكربت نفسه
- ✅ `shorts_pipeline.py`: مسار منفصل ومبسّط للشورت فقط
- ✅ جدولة النشر بنفس التوقيت المتفق عليه سابقاً: 04:00 بتوقيت بغداد يومياً

## ⚠️ هذا إصدار MVP — اقرأ قبل التشغيل

- تم بناء **الهيكل الكامل والمنطق الكامل** لكل خطوة، لكن الكود **لم يُختبر على
  بيانات حقيقية** (لا يوجد وصول من بيئة البناء لـ Gemini API أو YouTube API
  الفعلية لاختبارها مباشرة). **من المتوقع تحتاج تعديلات بسيطة أثناء أول تشغيل
  فعلي** (أسماء حقول قد تختلف، تنسيقات استجابة API قد تتغير بنسخ لاحقة).
- ابدأ بتشغيل `workflow_dispatch` يدوياً لكل ملف بـ `.github/workflows/` بدل
  الاعتماد على الجدولة التلقائية أول مرة، لمراقبة الأخطاء مباشرة بسجل GitHub Actions.
- كما اتفقنا: **ابدأ بمسار الشورتس فقط** أول أسبوع (أخف على الموارد)، وفعّل
  `daily_pipeline.yml` (الفيديو الطويل) لاحقاً بعد التأكد من استقرار البقية.

## البنية

```
scripts/           كل منطق بايثون (trend/script/tts/seo/publish/...)
remotion/           مشروع المونتير (Ken Burns + كابشن متزامن)، Node.js
.github/workflows/  الجدولة (trend scan / daily pipeline / cleanup / backup)
SECRETS.md          قائمة كاملة بكل الأسرار المطلوبة وطريقة الحصول عليها
```

## التثبيت (خطوات أولية)

1. ارفع هذا المجلد كـ repo **عام (Public)** على GitHub (لتفادي حد دقائق
   GitHub Actions الشهري للـ private repos)
2. أنشئ جدول Google Sheets بالأوراق الأربعة المذكورة بـ SECRETS.md
3. أضف كل الأسرار بـ Repo Settings → Secrets → Actions (راجع SECRETS.md بالتفصيل)
4. شغّل `Trend Scan` يدوياً من تبويب Actions للتأكد إنه يقرأ الترندات صح
5. شغّل `Daily Video Pipeline` يدوياً وراقب السجل خطوة بخطوة

## أهم نقطة: السيو (`scripts/seo_optimizer.py`)

هذا أهم ملف بالمشروع حسب طلبك. يجمع تحليل أفضل 10 منافسين حالياً على نفس
الموضوع (بدون نسخ حرفي — تحليل نمط فقط) مع قواعد SEO الصارمة (طول العنوان،
هيكلة الوصف، الفصول، الوسوم) ليبني بيانات وصفية بأعلى مستوى احترافية ممكن.
إذا لاحظت أداء ضعيف بالبحث لاحقاً، هذا أول ملف تراجعه وتحسّنه بناءً على بيانات
حقيقية من YouTube Analytics.

## ضمان دقة 1080p

- `remotion/src/Root.tsx`: الـ Composition مضبوط `width=1920 height=1080` مباشرة بالكود
- `scripts/publish.py` → `_verify_1080p()`: يفحص الفيديو فعلياً بـ ffprobe **قبل
  الرفع** ويرفض الرفع إذا الدقة أقل من 1080p بأي سبب غير متوقع (طبقة حماية مضاعفة)

## ما لم يُبنَ بعد بهذا الإصدار (تحتاج تطوير إضافي لاحقاً)

- ربط فعلي لتحميل ملف `captions.json` داخل `MainVideo.tsx` (حالياً مصفوفة فارغة
  توضيحية بالبنية — يحتاج `staticFile` أو تمرير المحتوى مباشرة بالـ props)
- سكربت الحصول على `YOUTUBE_OAUTH_REFRESH_TOKEN` لأول مرة (خطوة يدوية تفاعلية
  لمرة واحدة، مذكورة بـ SECRETS.md لكن السكربت نفسه غير مرفق)
- تقييم بصري فعلي لاختيار أفضل نسخة thumbnail (حالياً يختار أول نسخة دائماً
  كمسار آمن، راجع `pick_best_thumbnail` بـ `thumbnail_generator.py`)

## خطة التوسّع لاحقاً للفيديو الطويل

كل الكود جاهز (`daily_pipeline.py`, `long_video_pipeline_FUTURE.yml`,
Composition `LongVideo` بـ Remotion). لما تجهز:
1. راقب أداء الشورتس أسبوعين-ثلاثة (retention، مشتركين، أي تحذيرات من يوتيوب)
2. افتح `long_video_pipeline_FUTURE.yml` وفعّل سطر الـ `schedule` (احذف `#`)
3. لا حاجة لتعديل أي كود آخر — كل شيء متوافق مسبقاً
